# Product

## Register

product

## Users

主要用户是需要发现 LLM、理解模型能力并比较 API 接入渠道的开发者与技术决策者。他们通常在选型、成本评估或接入前核对价格与供应商信息，需要快速扫描、筛选和追溯数据来源。

## Product Purpose

LLM Info 将分散的模型、供应商和原始报价聚合为可信的模型目录。成功体验应让用户先找到规范模型，再查看能力与上下文信息，最后在同一模型内比较多个供应商的原生 USD/CNY 报价，而不误把缺失价格、免费报价或不同渠道混为一谈。

## Brand Personality

冷静、专业、透明。界面应像成熟的开发者工具：信息密度高但层级清楚，视觉语言克制，数据来源和缺失状态诚实可见。

## Anti-references

- 不做营销型 hero、口号先行或首屏宣传页。
- 不使用装饰性卡片堆叠、夸张渐变、霓虹发光或无功能价值的动效。
- 不把模型目录做成促销 marketplace，不使用“最佳”“最强”等缺少数据依据的推荐语。
- 不为追求新奇而改造表格、筛选器、弹框等标准产品交互。

## Design Principles

1. 先模型，后渠道：保持规范模型与渠道模型的层级清晰。
2. 扫描优先：列表和对比表应支持快速比较、排序和筛选。
3. 数据透明：价格来源、更新时间、币种与缺失状态必须可追溯。
4. 熟悉胜过新奇：使用成熟工具中可预测的导航、表格和弹框模式。
5. 克制表达：颜色只服务状态、选择与关键操作，不用于装饰。

## Accessibility & Inclusion

以 WCAG 2.2 AA 为基线。文本和控件满足对比度要求；焦点状态清晰；价格缺失、免费、官方渠道等状态不能只依赖颜色表达；交互目标尺寸合理；支持键盘操作和 reduced motion。
